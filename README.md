## About me

I'm Dani, a developer from Uruguay focused on backend architecture, runtime analysis, observability, automation and product-grade tooling.

I like systems that can be inspected, questioned and improved. If a service makes a decision, I want to understand why it happened, what context existed, what evidence supported it and how the behavior can be replayed later.

My work is centered around modular backend systems, request analysis, event pipelines, privacy-aware telemetry, operational checks and documentation that reflects how the system actually behaves.

Some of my current work is private because it is being built as product-grade tooling, not as a public demo.

## How I build

I prefer systems that are:

- explainable;
- privacy-aware;
- modular;
- testable;
- realistic under degraded conditions;
- honest about uncertainty;
- designed for real operational use.

A good system should not only ask:

```txt
Did something happen?
```

It should also ask:

```txt
Why did it happen?
What evidence supports it?
Can the decision be reviewed later?
What happens if part of the runtime context is unavailable?
What is the safest next step?
```
